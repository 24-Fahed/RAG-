Retrieval-Augmented Generation (RAG) is a technique that combines information retrieval with text generation. Instead of relying solely on the knowledge encoded in a language model's parameters, RAG systems first retrieve relevant documents from an external knowledge base, then use those documents as context for generating an answer.

The typical RAG pipeline consists of several stages. First, a classifier determines whether the user's query requires external knowledge retrieval. If retrieval is needed, the system generates a hypothetical answer using HyDE (Hypothetical Document Embeddings) to improve search quality. Next, the system performs both dense retrieval using vector similarity search in Milvus and sparse retrieval using BM25 keyword matching. These results are combined through hybrid fusion, which weights dense and sparse scores using a configurable alpha parameter.

After retrieving candidate documents, a reranking model such as MonoT5 or BGE sorts them by relevance. The top documents are then repacked using strategies like sides ordering, which places the most relevant documents at the beginning and end of the context to address the lost-in-the-middle phenomenon. A compression model like Recomp then reduces the context length while preserving key information. Finally, a large language model generates the answer based on the compressed context.
